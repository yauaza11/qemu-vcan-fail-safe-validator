### protocol.h

typedef struct{
    uint8_t seq_num;
    uint8_t main_aps;
    uint8_t sub_aps;
    uint8_t checksum;

여기서 __attribute__((packed)) 이걸 적어서 구조체에서 패딩하는것을 막는다고 한다! 그래서 8*4 = 32bit = 4byte로 고정한다고 함

### node_a_sender.c 작성
- 오늘의 어려움 : 초기화 코드가 너무 복잡한데 대체이걸 어떻게 내가 알아서 작성하라는거지? 였다 방법은 github에 can-utils라는게 있더라 여기에 can을 위한 코드가 많은데 cansend.c에서 초기화 코드를 그대로 가져다가 넣어버렸다. can fd나 can xl에 해당하는 내용은 주석처리하였다.
- can_dlc는 data length code이며 데이터를 효율적으로 보내기 위해 사용 

### 추가사항
- if, ifr : interface request 의 준말이다
- can_dlc는 data length code라 한다
- **can_id는 그저 우선순위만 의미있는게 아니다.**
    - Classic CAN에서는 ID가 11비트가 있다.(0x000 - 0x7FF)
    - CAN ID [3bit : 우선순위][3bit : 기능/도메인][5bit : 송신제어기 ID]
        - 우선순위
        - 기능/도메인 : ex.주기적인 데이터인지 고장 진단용 데이터인지 구분
        - 송신제어기 ID : 어떤 제어기가 쏜 데이터인가?
