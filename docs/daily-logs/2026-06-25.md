### 궁금해서 알아본것

- perror가 뭔데
    - 리눅스 시스템콜 에러 띄워주는것이다.
- 내가 vcan을 쓰기 위해 socketCAN쓰고 뭐 그랬는데 이게 소켓통신인가? 그럼 다른 시리얼 통신도 유사한 방법으로 리눅스 통신을 하는건가? 
    - 통신방법은 크게 두 가지로 나뉜다. (시리얼 통신, 네트워크 통신)
        - 시리얼 통신(I2C, SPI, UART)
            1. open()
            2. ioctl()
            3. write() / read()
            4. close()
        - 네트워크 통신(CAN, Ethernet)
            1. socket()
            2. ioctl()
            3. bind() 
            4. read() / write() / recv() / send()
            5. close()
            - 네트워크 통신은 ifreq를 쓰는데 이건 누가 보낸건지, 누가 받는지 써져있다. 뭐 메타 데이터라고 해도 될 듯.
