# qemu-vcan-fail-safe-validator

# QEMU vECU 기반 멀티스레드 CAN 통신 및 Watchdog 연동 Fail-Safe 검증
> **Fault Injection(결함 주입)을 통한 가상 전장 소프트웨어 강건성 및 비상 제어 시스템 검증**
---
## 1. 프로젝트 목적
차량 주행 중 제어기 오류나 통신 두절이 발생했을 때, 시스템이 안전 상태로 진입시키는 Fail-Safe 매커니즘을 구현해보고자 함
본 프로젝트는 QEMU 에뮬레이터, Linux vCAN환경을 활용하여 가상 제어기(vECU) 네트워크를 구축.
런타임 중 의도적으로 시스템 결함을 주입하여, Watchdog 타이머를 활용해 이를 실시간 감지 및 방어하는 시스템을 정량적으로 검증.
---
## 2. 시스템 아키텍처
하드웨어 결함 및 무한 루프 상황에서도 시스템 감지 능력을 잃지 않기 위해 멀티스레드 구조로 제어기 설계
- **Node A (가상 송신 제어기)** : 주기적으로 가상 차량 제어 데이터를 CAN 버스('vcan0')로 송신.
- **Node B (가상 수신 제어기)** : 멀티스레드 기반 방어 아키텍처 적용.
  - **Thread 1 (통신/제어 스레드)** : CAN 패킷 수신 및 정기 제어 출력 연산 수행.
  - **Thread 2 (Fail-Safe 스레드)** : 가상 Watchdog 타이머를 독립적으로 구동하며, Thread 1의 생존 여부 및 통신 주기 상태를 실시간 감시, 오류 감지 시 즉각 출력을 0으로 차단하고 비상 신호 송출.
---

## 4. 핵심 정량적 검증 지표 (수치화 목표)

### [METRIC 1] Fail-Safe 진입 타이밍 검증 (Timing)
- **측정 지표:** 결함 주입 시점 시점($T_{fault}$)부터 Fail-Safe 모드 진입(제어 출력 0) 및 비상 CAN 메시지(ID: 0x000) 송출 완료 시점($T_{safe}$)까지의 런타임 간격 ($\Delta T$).
- **검증 방법:** 의도적인 무한 루프 주입 후 Watchdog 타임아웃 감지 속도를 리눅스 타임스탬프 기반 마이크로초($\mu s$) 단위 측정하여 안전 데드라인 보장 검증.

### [METRIC 2] 네트워크 부하율 및 지연 시간 검증 (Network)
- **측정 지표:** 에러 버스트 패킷 발생 시 가상 CAN 버스 부하율(Bus Load, %) 및 최우선순위 메시지 전송 지연 시간(Latency, $ms$).
- **검증 방법:** 'canbusload' 툴을 사용하여 임계 버스 부하 환경을 강제 조성한 후, 고우선순위 제어 신호가 지연 없이 타겟 데드라인 이내에 도달하는지 추적.

### [METRIC 3] 코드 신뢰성 정적/동적 검증 (Quality)
- **측정 지표:** MISRA-C:2012 코딩 가이드라인 위반 개수, 'gcov' 기반 예외 처리 분기 커버리지(Branch Coverage, %).
- **검증 방법:** `cppcheck` 정적 분석을 CI 자동화 파이프라인(GitHub Actions)에 연동하여 위반 개수 0건 달성 및 모든 에러 분기문 실행율 100% 만족 검증.
